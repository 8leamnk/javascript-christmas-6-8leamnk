# 미션 - 크리스마스 프로모션

## 🔍 프로젝트 설명

> 12월 이벤트를 위해 비즈니스 팀에서 요청한 개발을 완료해야 한다.

### 프로젝트 상세 설명

#### 이벤트 기간

- 2023.12.1 ~ 2023.12.31

#### 이벤트 목표

1. 고객들이 혜택을 많이 받는다는 것을 체감한다.
2. 12월에 지난 5년 중 최고의 판매 금액을 달성한다.
3. 이벤트 참여 고객의 5%가 내년 1월 새해 이벤트에 재참여한다.

#### 사용자 주문 조건

- 음료만 주문할 수 없다.
- 메뉴는 한 번에 최대 20개까지만 주문할 수 있다.

#### 이벤트 적용 조건

- 총주문 금액 10,000원 이하는 적용되지 않는다.

#### 이벤트 내용

##### 할인

1. 날짜에 따라 할인이 달라진다.

- 크리스마스 디데이 할인: 2023.12.1 ~ 2023.12.25
- 평일 할인(일요일 ~ 목요일): 2023.12.1 ~ 2023.12.31
- 주말 할인(금요일, 토요일): 2023.12.1 ~ 2023.12.31
- 특별 할인(매주 일요일, 크리스마스 당일): 2023.12.1 ~ 2023.12.31

2. 가격에 따라 증정 이벤트가 발생한다.

- 증정 이벤트: 총주문 금액이 12만 원 이상일 때, 샴페인 1개 증정

##### 배지 부여

1. 배지 부여 기준: 총혜택 금액

2. 배지를 부여받을 수 있는 조건과 종류

- 5천 원 이상: 별
- 1만 원 이상: 트리
- 2만 원 이상: 산타

### 프로젝트 기능 요약

1. 사용자는 12월 중 식당 예상 방문 날짜를 입력한다.

- 예: 26

2. 사용자는 주문할 메뉴와 해당 메뉴의 개수를 입력한다.

- 예: 해산물파스타-2,레드와인-1,초코케이크-1

3. 사용자는 우테코 식당에서 받을 이벤트 혜택 내역을 미리 볼 수 있다.

### 프로젝트 핵심 기능

> 총혜택 금액을 구하는 것

## 📝 구현할 기능 목록

### 소개 출력 기능

- 맨 처음에 인사와 소개를 출력한다.

```
안녕하세요! 우테코 식당 12월 이벤트 플래너입니다.
```

### 날짜 입력 기능

- 사용자에세 12월 중 식당 예상 방문 날짜를 입력 받는다.
  ```
  12월 중 식당 예상 방문 날짜는 언제인가요? (숫자만 입력해 주세요!)
  ```
- `Console.readLineAsync`를 활용한다.
- 사용자가 입력한 날짜를 반환한다.

### 사용자가 입력한 날짜 유효성 검사 기능

- 예외가 발생하는 경우는 아래와 같다.
  1. 숫자가 아닌 경우
  2. 1 이상 31 이하의 숫자가 아닌 경우
  ```
  [ERROR] 유효하지 않은 날짜입니다. 다시 입력해 주세요.
  ```
- ❗️ 에러 메시지를 출력하고 해당 부분부터 입력을 다시 받아야 한다.

### 사용자에게 입력받은 날짜를 받아오는 기능

- 사용자의 방문 예상 날짜를 `12월 n일` 형태로 반환한다.
- `12월 n일에 우테코 식당에서 받을 이벤트 혜택 미리 보기!`문구를 출력하기 위해 필요하다.

### 메뉴 입력 기능

- 메뉴를 입력 받는다.
  ```
  주문하실 메뉴와 개수를 알려 주세요. (e.g. 해산물파스타-2,레드와인-1,초코케이크-1)
  ```
- `Console.readLineAsync`를 활용한다.
- 사용자가 입력한 메뉴를 반환한다.

### 사용자가 입력한 주문 내역의 유효성 검사 기능

- 예외가 발생하는 경우는 아래와 같다.
  1. 메뉴판에 없는 메뉴를 입력하는 경우
  2. 메뉴의 개수가 1 이상의 숫자가 아닌 경우
  3. 메뉴 형식이 예시와 다른 경우
  4. 중복 메뉴를 입력한 경우(e.g. 시저샐러드-1,시저샐러드-1)
  ```
  [ERROR] 유효하지 않은 주문입니다. 다시 입력해 주세요.
  ```
  5. 모든 메뉴 개수의 합이 20개 이상인 경우
  ```
  [ERROR] 메뉴는 한 번에 최대 20개까지만 주문할 수 있습니다. (e.g. 시저샐러드-1, 티본스테이크-1, 크리스마스파스타-1, 제로콜라-3, 아이스크림-1의 총개수는 7개)
  ```
  6. 음료만 주문했을 경우
  ```
  [ERROR] 음료만 주문 시, 주문할 수 없습니다.
  ```
- ❗️ 에러 메시지를 출력하고 해당 부분부터 입력을 다시 받아야 한다.

### 주문 메뉴의 문자열 변환 기능

- 주문한 메뉴 데이터를 가지고 문자열로 변환한다.
- 변환된 문자열과 메뉴를 반환한다.

### 미리보기 안내 문구 출력하는 기능

- 12월 n일에 우테코 식당에서 받을 이벤트 혜택 미리 보기! 문구를 출력한다.
- 사용자에게 입력받은 날짜를 활용하여 출력한다.

```
12월 3일에 우테코 식당에서 받을 이벤트 혜택 미리 보기!
```

### 사용자가 입력한 주문 메뉴를 출력하는 기능

- 주문 메뉴와 개수를 각각 출력한다.
- 순서는 자유롭게 출력이 가능하다.
- `Console.Print`를 활용한다.

```
<주문 메뉴>
티본스테이크 1개
바비큐립 1개
초코케이크 2개
제로콜라 1개
```

### 할인 전 총주문 금액을 계산하는 기능

- `Map()`을 활용하고 `reduce()`를 사용한다.
- 총주문 금액을 반환한다.

### 총주문 금액을 출력하는 기능

- 계산된 총주문 금액을 출력한다.
- 원 단위로 출력한다(통화 단위 적용).
- `Console.Print`를 활용한다.

```
<할인 전 총주문 금액>
142,000원
```

### 증정 메뉴를 계산하는 기능

- 할인 전 총주문 금액이 12만 원 이상일 때, 샴페인 1개 증정
- 증정 이벤트의 **금액**을 반환한다.

### 증정 메뉴를 출력하는 기능

- `0` 또는 `25000`의 값으로 반환된 값을 사용한다.
- `0`이면 `없음`으로, `25000`이면 `샴페인 1개`로 출력한다.
- 개 단위로 출력한다.
- `Console.Print`를 활용한다.

```
<증정 메뉴>
샴페인 1개
```

### 혜택 내역과 총혜택 금액을 계산하는 기능

#### 혜택 내역을 계산

- 계산 기준
  - 크리스마스 디데이 할인: 2023.12.1 ~ 2023.12.25
  - 평일 할인(일요일 ~ 목요일): 2023.12.1 ~ 2023.12.31
  - 주말 할인(금요일, 토요일): 2023.12.1 ~ 2023.12.31
  - 특별 할인(매주 일요일, 크리스마스 당일): 2023.12.1 ~ 2023.12.31
- `if`와 `Map`을 활용한다.
- 혜택 내역을 반환한다.

#### 총혜택 금액을 계산

- 총혜택 금액 = 할인 금액의 총합 + 증정 메뉴의 가격
- 총혜택 금액을 반환한다.
- 혜택이 없어도, `0`으로 반환한다.

### 혜택 내역을 출력하는 기능

- 고객에게 **적용될 이벤트 내역만** 보여준다.
- 여러 개의 이벤트가 적용된 경우 출력 순서는 자유롭게 출력한다.
- 이벤트가 적용되지 않을 경우, `없음`으로 출력한다.
- 원 단위로 출력한다(통화 단위 적용).
- `Console.Print`를 활용한다.

```
<혜택 내역>
크리스마스 디데이 할인: -1,200원
평일 할인: -4,046원
특별 할인: -1,000원
증정 이벤트: -25,000원
```

### 총혜택 금액을 출력하는 기능

- 계산된 총혜택 금액을 출력한다.
- 원 단위로 출력한다(통화 단위 적용).
- `Console.Print`를 활용한다.

```
<총혜택 금액>
-31,246원
```

### 할인 후 예상 결제 금액을 계산하는 기능

- 할인 후 예상 결제 금액 = 할인 전 총주문 금액 - 할인 금액
- 할인 후 예상 결제 금액을 반환한다.

### 할인 후 예상 결제 금액을 출력하는 기능

- 계산된 할인 후 예상 결제 금액을 출력한다.
- 원 단위로 출력한다(통화 단위 적용).
- `Console.Print`를 활용한다.

```
<할인 후 예상 결제 금액>
135,754원
```

### 12월 이벤트 배지를 계산하는 기능

- **총혜택 금액**에 따라 이벤트 배지의 이름을 다르게 반환한다.
  - 5천 원 이상: 별
  - 1만 원 이상: 트리
  - 2만 원 이상: 산타
- `if`문을 활용한다(`else`를 사용하지 않는다.).

### 12월 이벤트 배지를 출력하는 기능

- 반환된 배지의 이름을 출력한다.
- 배지가 부여되지 않는 경우, `없음`으로 출력한다.
- `Console.Print`를 활용한다.

```
<12월 이벤트 배지>
산타
```

## 📌 프로젝트 검토 목록

> 기능 구현, 테스트를 하면서 아래 사항들을 계속 검토한다.

### 요구 사항 검토

- 기능 요구 사항, 프로그래밍 요구 사항, 과제 진행 요구 사항을 준수했는지 검토한다.
- 놓친 부분이 있는지, 이해를 잘못한 부분이 있는지 다시 한 번 검토한다.
  - 함수 또는 메서드
    - 15라인을 넘어간 함수(메서드)가 있는지 확인한다.
    - 한 가지 일만 하도록 구현했는지 검토한다.
    - indent depth가 3이 넘었는지 확인한다.
  - 핵심 로직을 구현하는 코드(비즈니스 로직)와 UI를 담당하는 로직을 구분했는지 확인한다.
- 수정하거나 삭제하면 안 되는 곳을 건드렸는지 검토한다.
  - `package.json`을 변경했는지 확인한다.
  - `InputView`, `OutputView` 객체를 활용하여 입력과 출력을 담당하는 객체를 별도로 구현했는지 확인한다.
  - 파일 **경로**는 변경 가능하다.
- 코딩 컨벤션을 제대로 준수했는지 검토한다.

### 피드백 검토

> 1주차, 2주차, 3주차 공통 피드백을 잘 적용했는지 검토한다.

- 핵심 기능이 포함된 동작 가능한 작은 것을 먼저 만들었는지 잊지 않는다. ⭕️
- 요구 사항 정리를 잘했는지 확인한다.
- 기능 목록을 다시 한 번 살펴본다.
- 발생할 수 있는 예외 상황에 대해 고민했는지 확인한다.
- 객체스럽게 사용했는지 확인한다(데이터를 꺼내지(get) 말고 메시지를 던지도록 했는지 확인한다.).
- 필드의 수를 최소화했는지 확인한다.
- if, for, while문 사이의 공백을 지켰는지 확인한다.
- 함수명, 메서드명, 상수명, 변수명 등 이름을 확인한다. (불용어 금지)
- 값을 하드 코딩했는지 확인한다.
- 객체를 생성하는 또다른 방법을 사용했는지 확인한다.
- 핵심 기능에 가까운 부분부터 작게 테스트를 만들었는지 확인한다.
- 테스트 코드를 리팩터링했는지 확인한다.
